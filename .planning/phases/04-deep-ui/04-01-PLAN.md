Step 1 — CSS (style.css): Add ripple styles and keyframes BEFORE the existing `/* ===== Animations ===== */` section (line 1356):

Add `.ripple-effect` class: `position: absolute; border-radius: 50%; transform: scale(0); animation: rippleExpand 0.6s ease-out forwards; pointer-events: none; background: rgba(255,255,255,0.35);`

Add `@keyframes rippleExpand`: `to { transform: scale(4); opacity: 0; }`

Step 2 — JS (app.js): Add a `createRipple(e)` function after the existing `spawnSparkles` function (around line 817). The function:
1. Gets the clicked element via `e.currentTarget`
2. Checks `if (el.disabled) return;`
3. Creates a `span` element, adds class `ripple-effect`
4. Gets button's `getBoundingClientRect()` to calculate relative position
5. Sets `const size = Math.max(rect.width, rect.height)`
6. Sets span `width` and `height` to `size + 'px'`
7. Sets span `left` to `(e.clientX - rect.left - size/2) + 'px'`
8. Sets span `top` to `(e.clientY - rect.top - size/2) + 'px'`
9. Appends span to `el`
10. Adds `setTimeout(() => span.remove(), 600)` to clean up

Step 3 — JS (app.js): Add a delegated event listener at the bottom of the file (near the existing `document.addEventListener('click', ...)` handler). Use event delegation on `document`:
`document.addEventListener('click', e => { const btn = e.target.closest('.btn, .opt-btn, .play-btn, .settings-chip'); if (btn) createRipple(e); });`

Note: The `.btn` class already has `position: relative; overflow: hidden;` (style.css:262-264), so the ripple will be clipped correctly. The `.play-btn` is `position: relative` (line 647) and `overflow: hidden` is not set — add `overflow: hidden` to `.play-btn` in CSS. For `.opt-btn`, it already has `position: relative` (line 816). For `.settings-chip`, add `position: relative; overflow: hidden;` if not present.

IMPORTANT: Do NOT add ripple to the document click handler that already exists for closing modals (app.js document click listener). Use a separate listener for ripple only.

Step 3 — JS (app.js): Add `flashScreen(color)` function after the `createRipple` function:

```
function flashScreen(color) {
    const overlay = $('flashOverlay');
    overlay.style.transition = 'none';
    overlay.style.background = color;
    overlay.style.opacity = '0.15';
    // Force reflow to apply the instant change
    overlay.offsetHeight;
    overlay.style.transition = 'opacity 0.4s ease-out';
    overlay.style.opacity = '0';
}
```

Step 4 — JS (app.js): Modify `handleAnswer()` (app.js:1116). In the `isCorrect` branch (after `btn.classList.add('correct')` at line 1128), add:

`flashScreen('var(--accent-error)');`

Note: CSS custom properties work in `style.background` when set via JS because the browser resolves them. Alternatively, use the hex values directly: `'#10b981'` for success, `'#ef4444'` for error.

IMPORTANT: The z-index of 50 places the flash above the sakura canvas (z-index: 0) and app content (z-index: 1) but below modals (z-index: 100+). This ensures the flash is visible but never blocks modals.
