  <action>
Step 1 — CSS (style.css): Add score digit animation styles after the existing `.score` class (around line 523):

`.score-anim` — `display: inline-flex; overflow: hidden; height: 1em; vertical-align: bottom;`

`.score-digit` — `display: inline-block; transition: transform 0.4s cubic-bezier(0.16, 1, 0.3, 1); will-change: transform;`

`@keyframes scoreRollUp` — `from { transform: translateY(100%); opacity: 0; } to { transform: translateY(0); opacity: 1; }`

`.score-digit.rolling` — `animation: scoreRollUp 0.4s cubic-bezier(0.16, 1, 0.3, 1) forwards;`

Step 2 — JS (app.js): Add `animateScore(element, newValue)` function after the `flashScreen` function (or after `spawnCelebration` if Plan 1 is not yet applied):

The function:
1. Gets the element reference
2. Gets `oldValue` from `element.textContent` (parse as integer, default 0)
3. If `oldValue === newValue`, return immediately
4. Converts `newValue` to string, pads with leading zeros to match `oldValue` length (minimum 1 digit)
5. Clears `element.innerHTML`
6. For each digit character in the new value string:
   a. Create a `span.score-digit`
   b. Set `textContent` to the digit
   c. Set `style.animationDelay` to `(index * 0.05) + 's'`
   d. Add class `rolling`
   e. Append to element
7. After animation completes (setTimeout 450ms), set `element.textContent = newValue` to ensure clean state

Step 3 — JS (app.js): Replace direct `textContent` assignments with `animateScore()` calls:

Replace line 1035 (`$('scoreText').textContent = gameState.score;` in loadQuestion) with:
`animateScore($('scoreText'), gameState.score);`

Replace line 1153 (`$('scoreText').textContent = gameState.score;` in handleAnswer) with:
`animateScore($('scoreText'), gameState.score);`

Replace line 1154 (`$('myScoreText').textContent = gameState.score;` in handleAnswer) with:
`animateScore($('myScoreText'), gameState.score);`

IMPORTANT: Keep the `textContent` assignment in loadQuestion as a fallback for the initial load (questionIndex === 0). Use `animateScore` only when the score actually changes. Check `if (gameState.questionIndex > 0)` before animating, or simply always animate (the function handles oldValue === newValue case).

IMPORTANT: The `.score` class has `font-variant-numeric: tabular-nums` which ensures consistent digit width — this is critical for the rolling animation to look smooth.
