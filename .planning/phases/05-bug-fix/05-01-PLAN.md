执行两个精确修改，恢复之前被commit 168eab6回退的已验证修复：

**修改1 — index.html line 458 ytPlayerContainer样式：**
将当前的内联style：
```
width:1px;height:1px;overflow:hidden;pointer-events:none;
```
替换为：
```
width:360px;height:200px;transform:scale(0.001);transform-origin:top left;pointer-events:none;
```
原因：手机浏览器要求iframe以全尺寸渲染才能初始化播放器。transform:scale(0.001)将玩家缩小到几乎不可见但保持全尺寸渲染。opacity:0方案在部分移动浏览器上也会阻止初始化（commit 3a8c256的实验结论）。

**修改2 — app.js line 427 YouTube搜索URL添加过滤参数：**
将：
```
https://www.googleapis.com/youtube/v3/search?part=snippet&type=video&videoCategoryId=10&maxResults=1&q=${encodeURIComponent(query)}&key=${key}
```
改为：
```
https://www.googleapis.com/youtube/v3/search?part=snippet&type=video&videoCategoryId=10&videoEmbeddable=true&videoSyndicated=true&maxResults=1&q=${encodeURIComponent(query)}&key=${key}
```
原因：videoEmbeddable=true过滤掉禁止嵌入的视频（会触发101/150错误如夢灯笼），videoSyndicated=true确保只返回可外部播放的视频。

不要修改其他任何地方。这两个修改与commit 545015c和3a8c256的最终修复方案一致。

在initSakura()函数中进行三项精确修改，均操作app.js：

**修改1 — Page Visibility API：在animate函数内添加可见性检查**
在animate函数最开头（在帧率限制检查之前）添加：
- 声明一个闭包变量 `let pageVisible = true;` 在function animate(ts)的外部（initSakura作用域内）
- 在initSakura()内部添加 `document.addEventListener('visibilitychange', () => { pageVisible = !document.hidden; });`
- 在animate(ts)的帧率限制检查之前添加：`if (!pageVisible) { requestAnimationFrame(animate); return; }`
此修改使得页面不可见时requestAnimationFrame继续调度但不执行任何绘制（clearRect/update/draw全部跳过），页面恢复可见时自动恢复。

**修改2 — 移动端shadowBlur降级：在Star.draw()中检测移动设备**
在Star.draw()中，ctx.shadowBlur = this.size * 2 (app.js line 1666) 之前添加移动端检测：
- 使用 `const isMobile = 'ontouchstart' in window || navigator.maxTouchPoints > 0;` 检测移动设备
- 移动端时使用 `ctx.shadowBlur = 0;`（完全禁用阴影模糊以最大节省GPU）
- 桌面端保持 `ctx.shadowBlur = this.size * 2;` 不变

**修改3 — D-09保证：不改变视觉效果**
- 不修改PETAL_COUNT、STAR_COUNT的值
- 不改变Petal或Star的任何其他属性（大小、颜色、速度、透明度）
- 不改变canvas的clearRect、resize逻辑
- 移动端shadowBlur降级仅影响星星的光晕模糊（移动端小屏幕上几乎不可见），其他视觉效果完全不变
  </action>
